# SafeStreet Amman

A machine learning pipeline that classifies every street segment in a target area of Amman, Jordan as wheelchair-accessible or inaccessible, and a web map viewer to explore the results.

Built as a graduation project at the King Abdullah II School of Information Technology.

---

## What it does

The notebook pulls Amman's street network from OpenStreetMap, combines it with slope data from an SRTM elevation model, and trains a CatBoost + XGBoost ensemble on human-verified labels and crowdsourced accessibility reports. The output is a GeoJSON file consumed by a single-page HTML map application.

**Model performance (v3):**

| Metric | Score |
|---|---|
| Accuracy | 0.859 |
| AUC-ROC | 0.882 |
| Evaluation | 5-fold spatial cross-validation |

| Version | Training data | Accuracy | AUC-ROC |
|---|---|---|---|
| v1 | 200 manual labels | 0.570 | 0.580 |
| v2 | + crowdsourced reports | 0.694 | 0.761 |
| v3 | + feature engineering, pseudo-labeling, ensemble | 0.859 | 0.882 |

## Features used

| Category | Features |
|---|---|
| Road type | highway rank, is major road, pedestrian score, is pedestrian zone |
| Accessibility tags | sidewalk, surface, smoothness, width, wheelchair tag |
| Slope | mean, max, variance, net slope, ADA fail flag, slope class, slope × length |
| Road properties | speed limit, bridge, tunnel, one-way, segment length, short street flag |
| Spatial | bearing, north-south orientation, centroid coordinates |
| Data quality | number of missing OSM tags |

Slope is the most important feature in Amman's hilly terrain. Segments with a maximum gradient above 8% are flagged as failing the ADA / ISO 21542 accessibility threshold.

| `source` | string | `manual`, `crowd`, or NaN |

## Technical details

**Algorithm:** Soft-voting ensemble — CatBoost (60%) + XGBoost (40%)

**Training data:** ~200 manually labeled segments + 136 crowdsourced reports + up to 600 high-confidence pseudo-labels

**Pseudo-labeling:** A base model assigns pseudo-labels to unlabeled segments with predicted probability < 0.15 (safe) or > 0.85 (unsafe). These are added to training with weight 0.35 to limit their influence relative to human-verified labels.

**Validation:** 5-fold spatial cross-validation on 1 × 1 km blocks. Spatial blocking prevents adjacent segments from leaking between train and test sets, which would inflate results due to spatial autocorrelation.

**Domain restriction:** Predictions are limited to a bounding box around the labeled area plus a 1 km buffer. This keeps labeling density realistic (~0.22%) and avoids predicting in uncharted regions.

**Threshold selection:** The decision threshold is chosen by maximising F1 on the out-of-fold precision-recall curve rather than defaulting to 0.5, to account for class imbalance.
